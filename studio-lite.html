🏗 Critical Minerals Records Studio Lite

Build a Critical Minerals Records Stage tool from a JSON or CSV metadata export - entirely in your browser. No Python, no install, no upload.

This is the no-code, in-browser path. Your data never leaves this page. The result is metadata-only by design (title, source, subjects, date, score) — it links out to each document and does not store or search body text. For messy formats (XML/PDF/scraped HTML) or a controlled taxonomy, use the full Python Critical Minerals Records Studio (app.py) instead.

1 Load your data

Choose a .json file (an array of records, or an object with an array inside) or a .csv file with a header row.

2 Map your fields

3 Scoring (optional)

Records Stage ranks cards by score and can hide low-scoring ones. The defaults give everything an equal score, so you can skip this entirely. Tune it to surface what matters.

Pick the records-stage.html from your toolkit-template repo. We embed your data into a copy of it — your original file is never modified.
